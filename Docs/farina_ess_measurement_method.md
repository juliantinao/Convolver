## 1. Overview and Purpose

The **Farina Exponential Sine Sweep (ESS)** method — also called the log sweep or chirp method — is a technique for measuring the complete acoustic impulse response (IR) of a system using a single measurement pass. The method simultaneously yields:

- The linear impulse response h₁(t) of the system (the main deliverable)
- Separate impulse responses for each harmonic distortion order (h₂, h₃, ... hₙ)

The system under measurement can be a room, a loudspeaker, a microphone chain, or any electroacoustic signal path. The method assumes the system is approximately **linear and time-invariant (LTI)**, but crucially tolerates mild nonlinearity without corrupting the linear result — a key advantage over Maximum Length Sequence (MLS) techniques.

The full pipeline is entirely **offline and non-real-time**: the sweep is first synthesised, then played and recorded, then processed. The JUCE implementation will therefore have two distinct operational modes: **playback/record mode** and **analysis mode**.

---

## 2. Physical and Mathematical Background

### 2.1 The System Model

We model the device under test (DUT) as a black box with one audio input x(t) and one audio output y(t). In the ideal linear, time-invariant case:

```
y(t) = x(t) ⊗ h(t) + n(t)
```

where `⊗` denotes convolution, h(t) is the unknown impulse response, and n(t) is additive noise. Our goal is to recover h(t) from the measured output y(t), given that we know x(t) exactly.

### 2.2 The Nonlinear Extension

Real systems (especially loudspeakers) introduce harmonic distortion. Farina models this with a Volterra series representation. The memoryless nonlinear stage produces powers of the input, which the subsequent linear reverberation then filters:

```
y(t) = n(t) + x(t)⊗h₁(t) + x²(t)⊗h₂(t) + x³(t)⊗h₃(t) + ... + xᴺ(t)⊗hₙ(t)
```

Each hₙ(t) is the impulse response corresponding to the N-th order harmonic distortion. The ESS method recovers all of these simultaneously, separated in time after deconvolution.

### 2.3 The Core Insight: Linear vs. Circular Deconvolution

Traditional methods (MLS) use **circular deconvolution** (achieved via FFT division with a periodic signal). This folds distortion artefacts back into the measurement window, contaminating the result.

The ESS method uses **linear deconvolution** — convolving the recorded response with a pre-computed inverse filter. This pushes all distortion products to time positions **before** the linear response peak, where they can be cleanly windowed away.

## 3. Why Exponential (Logarithmic) Sweep?

This is a critical design choice with deep consequences.

### 3.1 Frequency Coverage per Octave

In an exponential sweep, **equal time is spent in every octave**. This means:
- Low frequencies receive as much energy per octave as high frequencies.
- The resulting spectrum is coloured at +6 dB/octave (more energy at high frequencies when viewed per-Hz), but flat on an octave (logarithmic) basis.
- This produces a much higher signal-to-noise ratio (SNR) at low frequencies compared to a linear sweep.

### 3.2 Distortion Separation Property

When a log sweep passes through a nonlinear system, the Nth harmonic of the instantaneous frequency behaves as if it were generated by a copy of the sweep starting at a fixed time offset **before** the fundamental. After deconvolution with the inverse filter, each harmonic order collapses into its own impulse peak, appearing at a deterministic time delay prior to the linear response:

```
Δtₙ = T × ln(N) / ln(ω₂/ω₁)
```

Where:
- N = harmonic order (2, 3, 4, ...)
- T = total sweep duration in seconds
- ω₁ = start angular frequency (2π × f₁)
- ω₂ = end angular frequency (2π × f₂)

This means the 2nd harmonic appears at `Δt₂ = T × ln(2) / ln(f₂/f₁)` seconds before the linear peak, the 3rd harmonic at `Δt₃ = T × ln(3) / ln(f₂/f₁)` seconds before, and so on. These offsets are predictable and fixed, enabling clean separation by time-windowing.

### 3.3 SNR Advantage

Because the sweep duration T is long (typically 5–30 seconds), a large amount of energy is injected into the system. The deconvolution then compresses this energy back into a short IR. The theoretical SNR gain is approximately:

```
SNR gain ≈ 10 × log₁₀(fs × T)   [dB]
```

For a 10-second sweep at 48 kHz, this is approximately 56 dB. In practice, gains of 60 dB or more over a single-impulse measurement are routinely achieved.

## 4. The Complete Algorithmic Pipeline

The pipeline consists of exactly seven stages. Stages 1–2 happen before the measurement. Stage 3 is the hardware interaction. Stages 4–7 are offline computation.

---

### Stage 1: Parameter Definition

Before any signal is generated, the following parameters must be fixed. They must remain **identical** between signal generation and inverse filter construction.

| Parameter | Symbol | Typical Value | Notes |
|-----------|--------|---------------|-------|
| Sample rate | fs | 44100 or 48000 Hz | Must match audio hardware exactly |
| Start frequency | f₁ | 20 Hz | Lower limit of measurement range |
| End frequency | f₂ | 20000 Hz | Upper limit; must be < fs/2 |
| Sweep duration | T | 10–30 s | Longer = better SNR |
| Silence tail duration | T_silence | ≥ expected RT60 | Prevents time-aliasing |
| Bit depth | — | 24 bit | 16 bit introduces excessive quantisation noise |
| Number of averages | M | 1 (preferred) or 3–5 | See note below |

**Critical note on averaging:** Farina recommends a single, long sweep over multiple short averaged ones. Averaging requires the system to be time-invariant across multiple repetitions, which is rarely guaranteed in real rooms (air movement, temperature drift). A single 20-second sweep is preferable to four 5-second sweeps averaged. If averaging is used, only identical (coherent) repetitions should be summed.

**Derived parameters (compute these once at init):**

```
N_sweep  = round(T × fs)         // number of samples in the sweep
N_silence = round(T_silence × fs) // number of zero samples appended
N_total  = N_sweep + N_silence   // total playback buffer length
L        = T / ln(f₂/f₁)        // sweep rate constant (used in generation formula)
```

---

### Stage 2: ESS Signal Generation

The Exponential Sine Sweep is defined by the following equation (Farina 2000, eq. derived in Section 2):

```
x(n) = sin( 2π × f₁ × L × (exp(n / (fs × L)) - 1) )
```

where n is the discrete sample index from 0 to N_sweep - 1.

**Step-by-step construction:**

**2.1 Allocate a buffer** of length N_total, initialised to zero.

**2.2 Fill samples 0 to N_sweep-1** using the formula above. For each sample index n:

```
t = n / fs                                    // time in seconds
instantaneous_phase = 2π × f₁ × L × (exp(t/L) - 1)
x[n] = sin(instantaneous_phase)
```

This ensures the instantaneous frequency at time t is:

```
f_inst(t) = f₁ × exp(t / L)
```

which starts at f₁ when t=0 and reaches f₂ when t=T (by construction of L).

**2.3 Append zeros** from N_sweep to N_total - 1. This is the silence tail. It ensures the full system response (including late reverberation) is captured without time-aliasing wrapping back into the measurement window.

**2.4 Apply fade-in and fade-out windows (highly recommended):**

- **Fade-in:** Apply a half-Hann (raised cosine) window over the first ~50–100 ms of the sweep. This prevents a discontinuity at t=0 (the sweep begins mid-cycle and would otherwise create a spectral artefact).
- **Fade-out:** Apply a short half-Hann window over the last ~5–20 ms of the sweep. This prevents a step discontinuity at the end that creates ripple at the high-frequency end of the spectrum.

The fade-in duration should be at least one full period of f₁ (e.g., 50 ms for f₁=20 Hz). The fade-out can be very short.

**2.5 Amplitude normalisation:** Scale the buffer so its peak absolute value is just below full scale (e.g., multiply by 0.9 to leave 1 dB headroom). This avoids clipping on playback while maximising SNR.

**2.6 Store this buffer persistently.** Both the playback engine and the inverse filter construction algorithm need this exact buffer.

---

### Stage 3: Playback and Recording

This is the hardware interaction stage.

**3.1 Synchronous playback and record:** The ESS buffer must be played through the output channel(s) while simultaneously recording from the input channel(s). The key requirement is that playback and recording start at the same sample instant. JUCE's `AudioDeviceManager` and callback architecture naturally satisfy this requirement when operating in duplex mode — the same callback processes both input and output buffers at the same sample offset.

**3.2 Record buffer sizing:** Allocate a record buffer of at least N_total samples. In practice, allocate slightly more (e.g., N_total + some safety margin) in case of minor clock drift.

**3.3 Loopback reference (optional but recommended):** If the audio interface supports it, record the playback output on a second input channel simultaneously with the microphone signal. This provides a reference signal that captures any coloration introduced by the DAC and output electronics, enabling system self-calibration in Stage 4.

**3.4 Level calibration before measurement:** Before the actual sweep, verify that:
- The playback level does not clip the output stage.
- The recorded signal does not clip the input ADC.
- The recorded SNR is adequate (not dominated by noise floor).

This is typically done with a short test tone or a brief preview sweep at reduced gain.

**3.5 Record format:** Record at the same fs and bit depth as generation (24-bit integer or 32-bit float internally). Store the recorded buffer as a float array of length N_total.

**3.6 Multiple averages (if used):** Play and record M repetitions of the full N_total buffer. After recording, time-align each repetition (they should align exactly if clocking is synchronous) and sum them sample-by-sample. Divide by M. This reduces incoherent noise by sqrt(M) but does not help with coherent distortion.

---

### Stage 4: Inverse Filter Construction

The inverse filter f(t) is designed to satisfy:

```
x(t) ⊗ f(t) = δ(t - delay)
```

That is, convolving the original sweep with the inverse filter yields a delayed Dirac impulse. When the recorded response y(t) is then convolved with f(t), the result is the system's impulse response h(t).

**4.1 Time-reverse the sweep:**

Create a buffer `x_rev` of the same length N_sweep by reading x[] in reverse order:

```
x_rev[n] = x[N_sweep - 1 - n]    for n = 0 to N_sweep - 1
```

This operation corresponds to the "time reversal mirror" technique. In the frequency domain, time reversal conjugates the phase, effectively dividing phase by the sweep's phase response.

**4.2 Apply the amplitude compensation envelope:**

The ESS signal has a non-flat spectrum — specifically, it has 3 dB more energy per octave at higher frequencies (its power spectrum rises at +3 dB/octave). To make the inverse filter spectrally white (i.e., to achieve flat deconvolution across all frequencies), the time-reversed sweep must be amplitude-modulated by a function that decreases by **6 dB per octave** (equivalently, the amplitude scales as 1/f, or equivalently as 1/exp(t/L) in the time domain).

The compensation envelope in the time domain of the reversed signal is:

```
w[n] = exp(-n / (fs × L))        for n = 0 to N_sweep - 1
```

Note that since the reversed sweep starts at frequency f₂ and sweeps down to f₁, the envelope must **decrease** with n (high frequency at start gets lower amplitude, compensating for the higher energy the ESS allocated there).

Alternatively, this can be expressed directly: since in the reversed sweep index n=0 corresponds to the original t=T (frequency f₂), and n=N_sweep-1 corresponds to t=0 (frequency f₁):

```
f_inst_rev(n) = f₂ × exp(-n / (fs × L))
w[n] = f₁ / f_inst_rev(n) = exp(-n / (fs × L)) × (f₁/f₂) ... (scaled version)
```

The exact scaling constant does not affect the shape of the recovered IR — it only affects its amplitude. A common practical normalisation is:

```
w[n] = (1 / (N_sweep × f₁)) × exp(-n / (fs × L))
```

Or more simply, compute the raw amplitude-compensated reversed sweep and normalise so that `x_inv ⊗ x` has a peak of 1.0.

**The inverse filter** is then:

```
f[n] = x_rev[n] × w[n]           for n = 0 to N_sweep - 1
```

**4.3 Zero-pad the inverse filter** to length N_conv for convolution:

```
N_conv = next_power_of_two(N_sweep + N_total - 1)
```

Append zeros to f[] to reach this length. This is required for linear (non-circular) convolution via FFT.

**Important:** The inverse filter has exactly N_sweep samples (not N_total). The silence tail is not included — it is part of the recorded signal only, not the inverse filter.

---

### Stage 5: Deconvolution

Deconvolution is implemented as **linear convolution** of the recorded signal y[] with the inverse filter f[].

Linear convolution of two sequences of lengths A and B produces a result of length A + B - 1. In the frequency domain, linear convolution is implemented by zero-padding both signals to at least A + B - 1 samples (rounded up to a power of two), multiplying their FFTs, and taking the IFFT.

**5.1 Determine convolution length:**

```
N_y  = N_total                      // length of recorded signal
N_f  = N_sweep                      // length of inverse filter
N_out = N_y + N_f - 1              // minimum output length
N_fft = next_power_of_two(N_out)   // FFT size (power of two)
```

**5.2 Compute FFT of the recorded signal:**

Zero-pad y[] from N_y to N_fft samples, then compute:

```
Y[k] = FFT(y_zeropadded)          // complex spectrum, length N_fft
```

**5.3 Compute FFT of the inverse filter:**

Zero-pad f[] from N_f to N_fft samples, then compute:

```
F[k] = FFT(f_zeropadded)          // complex spectrum, length N_fft
```

**5.4 Multiply in the frequency domain:**

```
H[k] = Y[k] × F[k]               // element-wise complex multiplication
```

Note: This is multiplication, not division. The inverse filter was constructed analytically so that multiplication achieves deconvolution. This avoids the numerical instability of spectral division (dividing by near-zero values).

**5.5 Compute the IFFT:**

```
h_full[n] = IFFT(H[k])            // real-valued result, length N_fft
```

Take only the real part. The result h_full[] is the deconvolved full response buffer.

**5.6 Interpret the result structure:**

After deconvolution, h_full[] contains (from left to right in time):

```
[... h_N(t) ... h₃(t) ... h₂(t) ... h₁(t) ... silence tail ...]
```

- **h₁(t)** — the linear impulse response — is the **rightmost** (latest) significant peak complex.
- **h₂(t), h₃(t), ...** — the harmonic distortion IRs — appear at fixed time offsets **before** (to the left of) h₁.
- The linear IR appears at a time offset of approximately T seconds from the start of h_full[] (the convolution delay introduced by the inverse filter, which is N_sweep samples long).

The exact sample position of the linear IR peak is:

```
n_linear = N_sweep - 1             // approximately; may need to be found by peak search
```

Because the inverse filter length is N_sweep samples, the deconvolution compresses the sweep into a peak at lag N_sweep-1 in the output.

**5.7 Practical note on FFT implementation:**

For very long sweeps (e.g., 20s at 48kHz = 960,000 samples), the N_fft value will be approximately 2,097,152 (2^21). This is computationally expensive for a single FFT. Farina and others recommend using **partitioned (overlap-add or overlap-save) convolution** for large filters — the inverse filter is split into shorter blocks, each convolved independently and their outputs summed with appropriate overlap. This is memory-efficient and allows streaming but adds implementation complexity.

For an offline measurement tool where latency is irrelevant, a single large FFT is simplest and perfectly valid.

---

### Stage 6: Linear IR Extraction and Post-Processing

After deconvolution, the linear impulse response h₁(t) must be extracted from h_full[].

**6.1 Locate the linear IR window:**

The linear IR begins at approximately sample index `n_linear = N_sweep - 1`. To locate the peak precisely, search for the sample with maximum absolute value in h_full[] in the region around n_linear (within ±1000 samples). Set this as the IR start reference.

**6.2 Define the extraction window:**

Extract a window from the deconvolved buffer:

```
h₁[n] = h_full[n_linear + n]     for n = 0 to N_ir - 1
```

Where N_ir is the desired impulse response length in samples. A typical choice is:

```
N_ir = round(expected_RT60 × 3 × fs)   // e.g., 3× the expected reverberation time
```

For room acoustics, this is typically 1–5 seconds. For loudspeaker measurements in free field, milliseconds are sufficient.

**Caution:** Ensure the extraction window does not overlap with the harmonic distortion peaks that precede h₁. The closest preceding harmonic (2nd order) is at:

```
n_2nd_harmonic = n_linear - round(T × ln(2) / ln(f₂/f₁) × fs)
```

The extraction window for h₁ must start after this position.

**6.3 Apply a window function (optional but recommended):**

For room acoustics analysis, the extracted IR does not typically need windowing. For loudspeaker free-field measurements, apply a time gate:

- Fade-in: Short Hann half-window (e.g., 1 ms) at the start of h₁ to eliminate any pre-ringing.
- Fade-out: Hann half-window (e.g., 20–100 ms depending on expected response) at the end to taper the noise floor.

**6.4 Frequency response:**

The transfer function (frequency response) of the system is obtained by taking the FFT of h₁:

```
H₁[k] = FFT(h₁_windowed)
|H₁[k]| = magnitude response
arg(H₁[k]) = phase response
```

**6.5 Synchronous averaging (if multiple sweeps were recorded):**

If M repetitions were recorded and stored separately (before summation), average the extracted IR sections coherently:

```
h₁_avg[n] = (1/M) × Σᵢ h₁ⁱ[n]
```

This reduces incoherent noise by sqrt(M) ≈ a factor of sqrt(M) in amplitude or 10×log₁₀(M) dB.

### Stage 7: Harmonic Distortion Extraction (Optional)

This stage is only needed if distortion analysis is required (e.g., loudspeaker THD characterisation).

**7.1 Locate each harmonic IR:**

For harmonic order N (N = 2, 3, 4, ...), the time offset before the linear IR is:

```
Δtₙ = T × ln(N) / ln(f₂/f₁)     [seconds]
Δnₙ = round(Δtₙ × fs)            [samples]
```

The Nth harmonic IR is centred at:

```
n_Nth = n_linear - Δnₙ
```

**7.2 Extract each harmonic IR:**

```
hₙ[m] = h_full[n_Nth - N_ir_half + m]    for m = 0 to N_ir_window - 1
```

where N_ir_window is chosen to encompass the harmonic IR without overlapping adjacent orders.

**7.3 Compute harmonic distortion spectrum:**

Take the FFT of each hₙ to get its frequency response. The Nth-order harmonic distortion spectrum spans frequencies from f₁ to f₂/N (because the Nth harmonic of f₂/N equals f₂, the top of the sweep range).

**7.4 Compute THD as a function of frequency:**

For each frequency bin, Total Harmonic Distortion is:

```
THD(f) = sqrt(|H₂(f)|² + |H₃(f)|² + ... + |Hₙ(f)|²) / |H₁(f)|
```

expressed as a percentage or dB.

## 5. Numeric Conventions and Precision Notes

**5.1 Data type:** All internal computations should use 64-bit double precision floating point. The final IR output may be stored as 32-bit float. Avoid 16-bit integer internally — quantisation noise at 16 bits is significant for measurements.

**5.2 Phase accumulation:** When generating the sweep sample-by-sample, do **not** accumulate the phase angle naively across samples. Instead, compute the absolute instantaneous phase analytically for each sample from the closed-form formula. Accumulating phase incrementally risks floating-point drift over millions of samples.

```
// CORRECT — compute absolute phase each sample
phase[n] = 2π × f₁ × L × (exp((n/fs)/L) - 1)
x[n] = sin(phase[n])

// RISKY — incremental phase accumulation
phase += 2π × f_inst(n) / fs   // drift accumulates over N_sweep iterations
```

**5.3 FFT normalisation:** Ensure consistent FFT normalisation conventions throughout. A common convention is: forward FFT divides by N, inverse FFT does not (or vice versa). What matters is consistency — the same convention must be applied when computing Y, F, and H. The absolute amplitude of the extracted IR depends on this.

**5.4 DC and Nyquist bins:** In the frequency-domain multiplication H[k] = Y[k] × F[k], the DC bin (k=0) and the Nyquist bin (k=N_fft/2) are real-valued. Ensure your FFT implementation returns these correctly.

**5.5 Anti-aliasing:** The ESS signal itself is band-limited to [f₁, f₂] and will not alias if f₂ < fs/2. However, if a nonlinear system generates harmonics of f₂ that exceed fs/2, these alias back into the baseband. This can corrupt high-frequency harmonic measurements. If distortion analysis above fs/4 is needed, oversample the measurement.

## 6. Key Formulas Quick-Reference

### ESS Generation

```
L = T / ln(f₂/f₁)
x[n] = sin(2π × f₁ × L × (exp((n/fs)/L) - 1))
    where n = 0, 1, ..., N_sweep-1 and N_sweep = round(T × fs)
```

### Inverse Filter

```
f[n] = x[N_sweep - 1 - n] × exp(-n / (fs × L))
    where n = 0, 1, ..., N_sweep-1
```

### Deconvolution (frequency domain)

```
H[k] = FFT(y_zeropadded)[k] × FFT(f_zeropadded)[k]
h_full[n] = Re(IFFT(H)[n])
```

### Harmonic Distortion Time Offsets

```
Δtₙ = T × ln(N) / ln(f₂/f₁)    [seconds before linear peak]

N=2: Δt₂ = T × 0.693 / ln(f₂/f₁)
N=3: Δt₃ = T × 1.099 / ln(f₂/f₁)
N=4: Δt₄ = T × 1.386 / ln(f₂/f₁)
```

### Separation condition (harmonics must not overlap)

For distortion extraction to work correctly, the distance between adjacent harmonic peaks must be greater than twice the expected IR duration:

```
Δt₂ - Δt₃ = T × (ln(2) - ln(3)) / ln(f₂/f₁)   // spacing between 2nd and 3rd harmonic
```

For this to exceed the IR duration T_ir, we need:

```
T > T_ir × ln(f₂/f₁) / (ln(3) - ln(2))   ≈ T_ir × ln(f₂/f₁) / 0.405
```

For a room with T_ir = 2 s and f₂/f₁ = 1000 (20 Hz to 20 kHz), this requires T ≈ 34 seconds. For loudspeaker measurements in an anechoic setting (T_ir ≈ 5 ms), T ≈ 0.085 seconds suffices.

## 7. Common Pitfalls and How to Avoid Them

**Pitfall 1: Time aliasing**  
*Symptom:* The late part of the IR wraps around and appears as spurious noise before the direct peak.  
*Cause:* The silence tail T_silence is shorter than the system's reverberation time.  
*Fix:* Always set T_silence ≥ RT60 of the room. For unknown rooms, use 3–5 seconds as a safe default.

**Pitfall 2: Clock mismatch between playback and record**  
*Symptom:* The IR is slightly smeared or time-stretched compared to the true IR.  
*Cause:* The DAC and ADC clocks are not perfectly synchronised (common with two separate audio interfaces).  
*Fix:* Use a single audio interface in full-duplex mode. If two interfaces must be used, record the playback on a spare input channel and use it for time-alignment correction (correlation-based) in post-processing. Farina (2000) explicitly notes this is tolerated well by the ESS method but can be corrected.

**Pitfall 3: Clipping during playback**  
*Symptom:* The recovered IR has strong nonlinear artefacts even for linear systems; the distortion harmonics appear unusually large.  
*Cause:* The ESS signal is clipping at the DAC or amplifier.  
*Fix:* Reduce playback gain. Always calibrate level before the measurement.

**Pitfall 4: Incorrect inverse filter (wrong amplitude envelope)**  
*Symptom:* The frequency response of the deconvolved IR has a strong spectral tilt (+6 or -6 dB/octave slope).  
*Cause:* The amplitude compensation envelope was not applied, or was applied in the wrong direction (should decrease with time in the reversed sweep).  
*Fix:* Verify by self-testing: convolve the ESS with its own inverse filter. The result should be a near-perfect Dirac delta. Any spectral coloration indicates a filter generation error.

**Pitfall 5: Circular convolution contamination**  
*Symptom:* The deconvolved IR appears to have a low-level "pre-echo" or "ring" starting well before the direct peak.  
*Cause:* Zero-padding was insufficient, causing circular wrap-around in the FFT multiplication.  
*Fix:* Always zero-pad to at least N_y + N_f - 1 before FFT. Use next power of two for efficiency.

**Pitfall 6: Harmonic peaks overlapping the linear IR**  
*Symptom:* The linear IR is corrupted by low-frequency distortion products.  
*Cause:* Sweep too short relative to the system's RT60 for the chosen frequency range.  
*Fix:* Increase sweep duration T or reduce the frequency ratio f₂/f₁. Use the separation condition formula (Section 6) to verify before measuring.

**Pitfall 7: Poor SNR at low frequencies**  
*Symptom:* The IR looks clean at high frequencies but noisy below 100–200 Hz.  
*Cause:* Background noise (HVAC, traffic) dominates at low frequencies; the fade-in window was too short, introducing spectral leakage.  
*Fix:* Increase sweep length T, use a longer fade-in (at least one full cycle of f₁), and ensure the measurement environment is as quiet as possible.

## Summary: The Seven Steps in One Glance

```
BEFORE MEASUREMENT:
  [1] Set parameters: fs, f₁, f₂, T, T_silence
  [2] Generate ESS: x[n] = sin(2π f₁ L (exp(n/(fs·L))-1))  → apply fades → append silence
  [4*] Build inverse filter: f[n] = x[N-1-n] · exp(-n/(fs·L))
       (*can be built before or after recording, only needs the sweep parameters)

MEASUREMENT:
  [3] Play x[] through output, simultaneously record y[] from input

ANALYSIS:
  [5] Deconvolve: H[k] = FFT(y) × FFT(f) → h_full = IFFT(H)
  [6] Extract linear IR: window h_full around sample n_linear = N_sweep-1
  [7] (Optional) Extract harmonics at n_linear - Δnₙ for N=2,3,...
```
